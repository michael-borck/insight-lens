# Browsing & Filtering Units

The **Units** page lists every unit in your database. Click any row to open the [Unit Detail](unit-detail.md) page.

## Searching

A search box at the top of the page filters by unit code and unit title as you type. Matching is case-insensitive. Clear the search to go back to the full list.

## Filters

Alongside the search box, filters let you narrow the list by:

- **Year** — restrict to a particular academic year.
- **Semester** — Semester 1, Semester 2, or other periods present in your data.
- **Campus** — useful if you have multi-campus offerings.

Filters combine with the search box. For example, you can type `isys` and pick **Semester 1** to see only Information Systems units with a Semester 1 offering.

## Sorting

Column headers act as sort controls. Click a header to sort ascending, click again for descending. Common sorts:

- **Unit code** — alphabetical, the default.
- **Overall experience** — find the best and worst quickly.
- **Response rate** — spot units where the sample is too small to trust.
- **Most recent offering** — bring newly imported units to the top.

## Row actions

Each row shows the unit code, title, most recent overall experience, and response rate. Clicking the row opens the unit detail page for a full semester-by-semester breakdown, comments, and sentiment.

## Keeping the list tidy

If you find the list cluttered, remember that every row represents a unit — not a survey. A unit with ten imported offerings is still one row. To see all offerings for a unit, open its detail page.
